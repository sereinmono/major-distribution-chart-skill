```markdown
# Skill: 专业分流成绩可视化分析（通用版）

## 触发条件
当用户需要分析专业分流数据、制作分流可视化图表、或从两张 Excel（成绩排名表 + 分流结果表）中提取 CSV 和统计图时触发。

## 输入要求
- **成绩排名表** Excel：至少包含学生唯一标识列、成绩数值列；可选包含需剔除的荣誉/创新标记列
- **分流结果表** Excel：至少包含学生唯一标识列、录取专业/方向列

## 核心工作流程

### Step 1: 数据探索（必须先做）
读取两张 Excel，打印前 5 行和列名，向用户确认或自行推断：
1. **学生ID列**（如"一卡通号"、"学号"、"student_id"等）
2. **成绩列**（如"首修平均成绩"、"GPA"、"平均分"等）—— 确认是百分制还是绩点制
3. **专业列**（如"录取专业"、"分流结果"、"major"等）
4. **是否需要剔除的列**（如"学术创新"、"荣誉表彰"等标记列，默认删除非空行；若用户要求保留则跳过）

> 原则：不预设任何固定列名，每次根据实际 Excel 结构动态确定。

### Step 2: 数据清洗
- 统一学生ID格式：转为字符串，去除 `.0` 后缀和前后空格
- 若存在荣誉/创新标记列，删除该列非空（且非空字符串）的学生
- 对两张表按学生ID做 **inner join**，仅保留同时存在的记录
- 删除成绩或专业缺失的行
- 报告清洗后有效记录数

### Step 3: CSV 导出
导出两张 CSV（UTF-8 with BOM）：
- `score_table.csv`：字段 `student_id`, `score`（或用户指定的成绩字段名）
- `major_result.csv`：字段 `student_id`, `major`

### Step 4: 图表生成（三张必选图）

#### 图1：堆叠密度图（density_plot.png）
- 横轴：成绩（使用用户数据中的实际成绩字段）
- 纵轴：Density（核密度估计）
- 按**实际发现的所有专业**分别绘制 KDE 曲线，堆叠填充
- 每个专业分配不同颜色，图例标注专业名称
- 标题、轴标签使用中文或中英双语
- dpi = 300

#### 图2：冲积图（alluvial.html + alluvial.png）
- 将全体学生成绩按 **等频四分位数** 划分为 Q1(低)、Q2、Q3、Q4(高)
- 左侧节点：4 个成绩分段（标签标注分数范围 + 人数 + 占比）
- 右侧节点：所有实际发现的专业（标签标注人数 + 占比）
- 条带宽度 = 流向人数，颜色按**目标专业**着色
- 使用 Plotly Sankey 输出交互式 HTML；若 kaleido 可用则同时输出静态 PNG
- 标题反映实际成绩字段名称

#### 图3：散点图（rank_scatter.png）
- 横轴：`overall_rank / N_total × 100%`（越小越靠前）
- 纵轴：`(1 − major_rank / N_major) × 100%`（越大越靠前）
- 按专业分色绘制散点（alpha=0.5）
- **不绘制任何趋势线**
- 添加参考线 `y = 100 − x`（灰色虚线），在图例中标注为 `y = x`
- **纵轴刻度标签翻转**：图像保持原样，仅将刻度文字反转为 100%→0%（即顶部显示 0%，底部显示 100%）
- dpi = 300

### Step 5: 验证与交付
检查以下 5 个文件是否全部生成：
- `score_table.csv`
- `major_result.csv`
- `density_plot.png`
- `alluvial.html`（+ `alluvial.png` 若成功）
- `rank_scatter.png`

若用户要求，为每张图提供一段简短的文字注解。

## 关键原则与自由度
| 决策点 | Agent 处理方式 |
|--------|---------------|
| 列名识别 | 先打印列名，自动匹配或询问用户，绝不硬编码 |
| 专业数量 | 动态识别所有唯一值，不限制为4个 |
| 颜色分配 | 根据实际专业数量动态生成调色板，保持专业间区分度 |
| 荣誉学生 | 默认删除，用户明确说"保留"时才保留 |
| 分流表表头 | 若第一行是合并标题，使用 `skiprows` 跳过 |
| 成绩字段语义 | 根据数值范围判断是百分制(0-100)还是绩点制(0-5)，图表标签如实反映 |
| 编码问题 | 使用 `openpyxl` 读取，CSV 输出 `utf-8-sig` 保证 Excel 兼容 |

## 依赖
pandas, openpyxl, matplotlib, seaborn, plotly, scipy, statsmodels, kaleido（可选）
```
